# Workspace Booking Codebase

This folder contains the runnable application code for the Workspace Booking MVP.

## Structure

```txt
apps/
  api/       NestJS backend
  web-final/ Next.js frontend
packages/
  shared/    shared types and constants
supabase/
  01_schema.sql
  02_auth_and_policies.sql
  03_seed.sql
  04_expand_workspace_types.sql
  05_demo_reset_bookings.sql
```

## Stack

- Monorepo tool: `npm workspaces`
- Frontend: Next.js
- Backend: NestJS
- Database/Auth/Storage/Realtime: Supabase

## Environment files

- Consolidated runtime env: `src code/.env`
- Consolidated env template: `src code/.env.example`

Note: Next.js reads environment variables from `apps/web-final/.env.local`. This file is automatically generated and kept in sync from the root `.env` file when running `npm run dev:final` or `npm run build:final`. Do not edit it manually or commit it.

## Current backend modules

- Auth token validation
- Roles guard
- Buildings CRUD
- Floors CRUD
- Workspaces CRUD and SVG mapping support
- Booking lifecycle and QR check-in

## Supabase SQL order

Run these files in Supabase SQL Editor when setting up a fresh project:

1. `supabase/01_schema.sql`
2. `supabase/02_auth_and_policies.sql`
3. `supabase/03_seed.sql`
4. `supabase/04_expand_workspace_types.sql`

Use `supabase/05_demo_reset_bookings.sql` only for local/demo cleanup. It resets bookings for the three demo accounts and creates a small non-active history so active booking quota is clear.

## Running locally

From the `src code/` directory:

```powershell
npm install
npm run dev:api
npm run dev:final
npm run test:api
npm run test:e2e:api
```

## Running with Docker

From the `src code/` directory:

```powershell
docker compose up --build
```

- Web app: `http://localhost:3002`
- API: `http://localhost:3001`
- Swagger docs: `http://localhost:3001/docs`

Read [DOCKER.md](/d:/money/1-Workspace booking/src code/DOCKER.md:1) for rebuild and token notes.

For full onboarding steps, read the root-level [SETUP.md](/d:/money/1-Workspace booking/SETUP.md:1).

## Frontend E2E note

The web-final E2E suite uses Playwright and expects:

- frontend running at `http://127.0.0.1:3002`
- backend running at `http://localhost:3001`
- shell variables such as `E2E_EMAIL` and `E2E_PASSWORD`, or the defaults in `apps/web-final/.env.example`
